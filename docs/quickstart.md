# SOC Quickstart (GitHub)

This guide installs the SOC stack using the Practikal IT Solutions SOC GitHub repository as the entry point.

## One-liner (all-in-one)
Run this on a fresh supported Linux host:

- `curl -fsSLO https://raw.githubusercontent.com/practikalaus/soc/main/installer/soc-quickstart.sh && sudo bash ./soc-quickstart.sh -a`

What it does:
- Downloads upstream baseline installer (Wazuh) for the pinned major stream (default `4.14`)
- Generates a branded local installer `./soc-install.sh`
- Runs `./soc-install.sh -a` (all-in-one install)
- Generates `soc-install-files.tar` (not `wazuh-install-files.tar`)

## Safer two-step (download then run)
If you prefer to inspect the script before running:

- `curl -fsSLO https://raw.githubusercontent.com/practikalaus/soc/main/installer/soc-quickstart.sh`
- `less soc-quickstart.sh`
- `sudo bash ./soc-quickstart.sh -a`

## Pin the baseline stream
To pin the baseline major stream:

- `curl -fsSLO https://raw.githubusercontent.com/practikalaus/soc/main/installer/soc-quickstart.sh && sudo bash ./soc-quickstart.sh --major 4.14 -a`

## Files created in the current directory
- `soc-install.sh` — generated branded installer
- `config.yml` — upstream config file copy (useful later for distributed installs)
- `soc-install-files.tar` — certificates/passwords bundle generated by the installer

## Notes / constraints
- Underlying packages/services/paths are still the baseline names for compatibility (this is intentional).
- For multi-node installs (separate indexer/server/dashboard), use the repo workflow described in the root README and the `installer/run-soc.sh` wrapper.

## If you see “already installed” errors
If you run the all-in-one quickstart (`-a`) on a host that already has the baseline components installed, the installer will refuse to proceed.

Options:
- Use a fresh host (recommended).
- If you intentionally want to wipe and reinstall, add `-o` / `--overwrite`:
	- `curl -fsSLO https://raw.githubusercontent.com/practikalaus/soc/main/installer/soc-quickstart.sh && sudo bash ./soc-quickstart.sh -a -o`

Warning: `-o/--overwrite` erases existing configuration and data.
